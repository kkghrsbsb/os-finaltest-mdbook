# 简答题打印抄写版

## 1. 何谓虚拟设备？SPOOLing 如何实现虚拟设备？

虚拟设备是指通过虚拟技术将一台独占设备变换为若干台逻辑设备，供若干个用户进程同时使用，经过虚拟技术处理后的设备称为虚拟设备。

SPOOLing 系统主要由输入井和输出井、输入缓冲区和输出缓冲区、输入进程和输出进程组成。输入时，输入进程把程序和数据从输入设备经输入缓冲区送入输入井，程序运行需要输入时直接从输入井读入内存。输出时，用户程序先把输出数据送到磁盘输出井，输出设备空闲时，再由输出进程经输出缓冲区送到输出设备。由于输入井、输出井是磁盘上的共享区域，多个进程可共享使用，因而把独占设备改造成了可共享的虚拟设备。

## 2. 为什么引入进程？进程与程序的区别和联系？

在多道程序并发执行时，各程序共享系统资源，并共同决定资源状态，程序执行会出现相互制约、走走停停、异步推进等动态特征。程序是静态概念，不能反映并发执行过程中的这些特征，所以引入进程来描述程序的动态执行过程。

区别：进程是动态的，程序是静态的；进程具有独立性，可以并发执行，程序本身不能并发执行；进程和程序没有一一对应关系；进程异步运行，会相互制约，程序不具备此特征。联系：进程不能脱离具体程序而存在，程序规定了相应进程所要完成的动作。

## 3. 单缓冲和双缓冲的区别及每块数据处理时间

设 `T` 为输入一块数据到缓冲区的时间，`M` 为缓冲区到用户区的传送时间，`C` 为 CPU 处理一块数据的时间。

单缓冲只有一个缓冲区，设备输入、缓冲区传送和用户处理的重叠程度较低，每块数据的平均处理时间为 `max(T, C) + M`。双缓冲有两个缓冲区，可使设备输入下一块数据与用户进程传送、处理上一块数据并行进行，每块数据的平均处理时间为 `max(T, C + M)`。双缓冲比单缓冲能更好地实现输入和处理的并行，提高 I/O 效率。

## 4. 什么是请求页式管理？能满足用户哪些需要？

请求页式管理是把内存和用户逻辑地址空间划分为同样大小的块，分别称为实页和虚页，通过页表建立虚页和实页的对应关系，并在地址变换时把逻辑地址转换为物理地址。程序运行时不要求全部页面一次装入内存，若访问的页已在内存中，则直接访问；若该页不在内存中，则产生缺页中断，把所需页调入内存，必要时进行页面置换。

请求页式管理能满足用户扩大内存空间的需要，实现内存与外存的统一管理；能提高内存利用率；不要求作业连续存放，能有效减少碎片问题。

## 5. 什么是虚拟存储器？特点是什么？为什么能扩大内存空间？

虚拟存储器是由操作系统提供的一个假想的特大存储器，是操作系统采用内外存交换技术，在逻辑上对物理内存的扩充。系统根据程序执行情况把页面或段换入、换出，用户感觉好像拥有比实际内存大得多的地址空间。

虚拟存储器的特点是虚拟性、离散性、多次性和对换性。虚拟性指逻辑容量大于实际内存容量；离散性指程序可以分散装入内存；多次性指作业不必一次全部装入；对换性指运行中可在内存和外存之间换入换出。

它能扩大内存空间，是因为程序具有局部性，运行时只需把当前需要的部分装入内存，其余部分保存在外存，需要时再调入。因此从逻辑上扩大了用户可用的内存空间。

## 6. 产生死锁的必要条件是什么？解决死锁常用哪些措施？

产生死锁的四个必要条件是：互斥条件、不剥夺条件、请求和保持条件、循环等待条件。互斥条件指某资源一次只能被一个进程使用；不剥夺条件指进程已获得的资源在未使用完前不能被强行夺走，只能由进程自己释放；请求和保持条件指进程已占有部分资源，同时又申请新的资源而等待；循环等待条件指系统中存在进程和资源的环形等待链。

解决死锁常用措施有：死锁预防，通过破坏死锁必要条件来预防死锁；死锁避免，在资源动态分配过程中防止系统进入不安全状态；死锁检测及解除，允许死锁发生，检测到死锁后通过撤销进程、剥夺资源等方法解除死锁。

## 7. 什么是多道程序设计技术？其特点是什么？

多道程序设计是指把多个程序同时存放在内存中，使它们同时处于运行过程中，这些作业共享处理器时间、外部设备以及其他系统资源。

多道程序设计技术的主要特点是多道、宏观上并行、微观上串行。多道是指内存中同时存放多道相互独立的程序；宏观上并行是指同时进入系统的多道程序都处于运行过程中；微观上串行是指在单处理器环境中，任一时刻只有一道程序占有 CPU，多道程序轮流占有 CPU，交替执行。

## 8. 简述 I/O 系统的层次模型及各层工作

I/O 系统的层次模型通常包括用户层 I/O 软件、设备独立性软件、设备驱动程序、中断处理程序和硬件层。

用户层 I/O 软件负责向用户程序提供 I/O 接口、库函数和系统调用。设备独立性软件负责统一设备命名、设备保护、缓冲管理、设备分配、错误处理，并实现逻辑设备到物理设备的转换。设备驱动程序负责把上层 I/O 请求转换成具体设备命令，设置设备控制器并启动设备。中断处理程序负责响应 I/O 中断、保存和恢复现场、进行中断处理并唤醒等待进程。硬件层由 I/O 设备、设备控制器、通道等组成，负责实际的数据传输。

## 9. 利用信号量描述前趋关系

用信号量描述前趋关系时，每一条前趋边设置一个信号量，初值为 0。若有前趋关系 `Si -> Sj`，则在前驱操作 `Si` 完成后执行 `V(aij)`，表示前驱已完成；在后继操作 `Sj` 开始前执行 `P(aij)`，表示必须等待前驱完成后才能继续执行。

若一个操作有多个前驱，则它开始前要对所有对应信号量执行 `P` 操作；若一个操作有多个后继，则它完成后要对所有对应信号量执行 `V` 操作。这样即可保证各操作按前趋图规定的次序执行。

## 10. 批处理、分时和实时操作系统各有什么特点？

批处理操作系统的主要特点是脱机、多道和成批处理。脱机是指用户提交作业后直到获得结果前几乎不再和计算机交互；多道是指从后备作业中选取多道作业调入内存并组织运行；成批处理是指操作员把作业组织成批，由操作系统自动调度执行。

分时操作系统的主要特点是多路性、交互性、独占性和及时性。多路性是指多个终端用户可以同时或基本同时使用计算机；交互性是指用户通过终端联机与程序交互；独占性是指每个用户感觉像独占计算机；及时性是指用户请求能在较短时间内得到响应。

实时操作系统的主要特点是及时性和高可靠性。及时性是指系统能及时响应外部事件，并在规定时间内完成处理；高可靠性是指系统必须安全可靠，避免因延误或错误造成严重后果。

## 11. 磁盘调度算法 FCFS、SSTF、SCAN

FCFS 是先来先服务算法，按照磁盘 I/O 请求到达的先后顺序进行调度。它简单、公平，但可能使磁头移动距离较大，平均访问时间较长。

SSTF 是最短寻道时间优先算法，每次选择距离当前磁头位置最近的 I/O 请求作为下一次调度对象。它能减少平均寻道时间，但可能使远离当前磁头的请求长期等待。

SCAN 是扫描算法，也称电梯算法。磁头沿一个方向移动，遇到 I/O 请求就处理，直到到达边缘或该方向无请求后再反向扫描。它既考虑距离又考虑移动方向，克服了 SSTF 可能产生饥饿的缺点。

计算磁头移动总量时，先写出各算法的访问顺序，再把相邻两次访问磁道号之差的绝对值相加。SCAN 题目必须注意磁头初始方向；若题目未给方向，应先说明自己的方向假设。
