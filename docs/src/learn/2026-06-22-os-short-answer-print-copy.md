# 简答题打印抄写版

## 1. 何谓虚拟设备？SPOOLing 如何实现虚拟设备？

1. 虚拟设备是指通过虚拟技术，将一台独占设备变换为若干台逻辑设备。
2. 它可供若干个用户进程同时使用。
3. 经过虚拟技术处理后的设备，称为虚拟设备。

SPOOLing 的实现：

1. SPOOLing 系统主要由三部分组成：
   输入井和输出井、输入缓冲区和输出缓冲区、输入进程和输出进程。
2. 输入时：
   输入进程把程序和数据从输入设备经输入缓冲区送入输入井；
   程序运行需要输入时，直接从输入井读入内存。
3. 输出时：
   用户程序先把输出数据送到磁盘输出井；
   输出设备空闲时，再由输出进程经输出缓冲区送到输出设备。
4. 由于输入井、输出井是磁盘上的共享区域，多个进程可以共享使用，
   因而把独占设备改造成了可共享的虚拟设备。

## 2. 为什么引入进程？进程与程序的区别和联系？

引入进程的原因：

1. 在多道程序并发执行时，各程序共享系统资源。
2. 程序之间会相互制约，执行过程会出现“走走停停”的动态特征。
3. 程序是静态概念，不能反映并发执行过程中的动态特征。
4. 因此引入进程，用来描述程序的动态执行过程。

进程与程序的区别：

1. 进程是动态的，程序是静态的。
2. 进程具有独立性，可以并发执行；程序本身不能并发执行。
3. 进程和程序没有一一对应关系。
4. 进程异步运行，会相互制约；程序不具备此特征。

进程与程序的联系：

1. 进程不能脱离具体程序而存在。
2. 程序规定了相应进程所要完成的动作。

## 3. 单缓冲和双缓冲的区别及每块数据处理时间

设：

1. `T` 为输入一块数据到缓冲区的时间。
2. `M` 为缓冲区到用户区的传送时间。
3. `C` 为 CPU 处理一块数据的时间。

单缓冲：

1. 单缓冲只有一个缓冲区。
2. 设备输入、缓冲区传送和用户处理的重叠程度较低。
3. 每块数据的平均处理时间为：`max(T, C) + M`。

双缓冲：

1. 双缓冲有两个缓冲区。
2. 设备输入下一块数据时，用户进程可以传送、处理上一块数据。
3. 每块数据的平均处理时间为：`max(T, C + M)`。
4. 双缓冲比单缓冲能更好地实现输入和处理的并行，提高 I/O 效率。

## 4. 什么是请求页式管理？能满足用户哪些需要？

请求页式管理：

1. 把内存和用户逻辑地址空间划分为同样大小的块。
2. 内存中的块称为实页，逻辑地址空间中的块称为虚页。
3. 通过页表建立虚页和实页的对应关系。
4. 地址变换时，把逻辑地址转换为物理地址。
5. 程序运行时，不要求全部页面一次装入内存。
6. 若访问的页已在内存中，则直接访问。
7. 若访问的页不在内存中，则产生缺页中断，把所需页调入内存。
8. 必要时，还要进行页面置换。

能满足的需要：

1. 扩大用户可用内存空间。
2. 实现内存与外存的统一管理。
3. 提高内存利用率。
4. 不要求作业连续存放，能有效减少碎片问题。

## 5. 什么是虚拟存储器？特点是什么？为什么能扩大内存空间？

虚拟存储器：

1. 虚拟存储器是由操作系统提供的一个假想的特大存储器。
2. 它是操作系统采用内外存交换技术，在逻辑上对物理内存的扩充。
3. 系统根据程序执行情况，把页面或段换入、换出。
4. 用户感觉好像拥有比实际内存大得多的地址空间。

特点：

1. 虚拟性：
   逻辑容量大于实际内存容量。
2. 离散性：
   程序可以分散装入内存。
3. 多次性：
   作业不必一次全部装入内存。
4. 对换性：
   运行中可以在内存和外存之间换入换出。

为什么能扩大内存空间：

1. 程序具有局部性。
2. 程序运行时，只需把当前需要的部分装入内存。
3. 其余部分保存在外存中，需要时再调入。
4. 因此从逻辑上扩大了用户可用的内存空间。

## 6. 产生死锁的必要条件是什么？解决死锁常用哪些措施？

产生死锁的四个必要条件：

1. 互斥条件：
   某资源一次只能被一个进程使用。
2. 不剥夺条件：
   进程已获得的资源在未使用完前不能被强行夺走，只能由进程自己释放。
3. 请求和保持条件：
   进程已占有部分资源，同时又申请新的资源而等待。
4. 循环等待条件：
   系统中存在进程和资源的环形等待链。

解决死锁的常用措施：

1. 死锁预防：
   通过破坏死锁产生的必要条件来预防死锁。
2. 死锁避免：
   在资源动态分配过程中，防止系统进入不安全状态。
3. 死锁检测及解除：
   允许死锁发生，检测到死锁后，通过撤销进程、剥夺资源等方法解除死锁。

## 7. 什么是多道程序设计技术？其特点是什么？

多道程序设计：

1. 多道程序设计是指把多个程序同时存放在内存中。
2. 使它们同时处于运行过程中。
3. 这些作业共享处理器时间、外部设备以及其他系统资源。

主要特点：

1. 多道：
   内存中同时存放多道相互独立的程序。
2. 宏观上并行：
   同时进入系统的多道程序都处于运行过程中。
3. 微观上串行：
   在单处理器环境中，任一时刻只有一道程序占有 CPU；
   多道程序轮流占有 CPU，交替执行。

## 8. 简述 I/O 系统的层次模型及各层工作

I/O 系统的层次模型通常包括：

1. 用户层 I/O 软件。
2. 设备独立性软件。
3. 设备驱动程序。
4. 中断处理程序。
5. 硬件层。

各层工作：

1. 用户层 I/O 软件：
   向用户程序提供 I/O 接口、库函数和系统调用。
2. 设备独立性软件：
   负责统一设备命名、设备保护、缓冲管理、设备分配、错误处理；
   并实现逻辑设备到物理设备的转换。
3. 设备驱动程序：
   把上层 I/O 请求转换成具体设备命令；
   设置设备控制器并启动设备。
4. 中断处理程序：
   响应 I/O 中断，保存和恢复现场；
   进行中断处理并唤醒等待进程。
5. 硬件层：
   由 I/O 设备、设备控制器、通道等组成；
   负责实际的数据传输。

## 9. 利用信号量描述前趋关系

基本方法：

1. 用信号量描述前趋关系时，每一条前趋边设置一个信号量。
2. 该信号量初值为 0。
3. 若有前趋关系 `Si -> Sj`：
   在前驱操作 `Si` 完成后执行 `V(aij)`；
   在后继操作 `Sj` 开始前执行 `P(aij)`。

一般规则：

1. `V(aij)` 表示前驱已经完成。
2. `P(aij)` 表示后继必须等待前驱完成后才能继续执行。
3. 若一个操作有多个前驱，则它开始前要对所有对应信号量执行 `P` 操作。
4. 若一个操作有多个后继，则它完成后要对所有对应信号量执行 `V` 操作。
5. 这样即可保证各操作按前趋图规定的次序执行。

## 10. 批处理、分时和实时操作系统各有什么特点？

批处理操作系统：

1. 主要特点是脱机、多道和成批处理。
2. 脱机：
   用户提交作业后，直到获得结果前几乎不再和计算机交互。
3. 多道：
   从后备作业中选取多道作业调入内存并组织运行。
4. 成批处理：
   操作员把作业组织成批，由操作系统自动调度执行。

分时操作系统：

1. 主要特点是多路性、交互性、独占性和及时性。
2. 多路性：
   多个终端用户可以同时或基本同时使用计算机。
3. 交互性：
   用户通过终端联机与程序交互。
4. 独占性：
   每个用户感觉像独占计算机。
5. 及时性：
   用户请求能在较短时间内得到响应。

实时操作系统：

1. 主要特点是及时性和高可靠性。
2. 及时性：
   系统能及时响应外部事件，并在规定时间内完成处理。
3. 高可靠性：
   系统必须安全可靠，避免因延误或错误造成严重后果。

## 11. 磁盘调度算法 FCFS、SSTF、SCAN

FCFS：

1. FCFS 是先来先服务算法。
2. 按照磁盘 I/O 请求到达的先后顺序进行调度。
3. 优点是简单、公平。
4. 缺点是可能使磁头移动距离较大，平均访问时间较长。

SSTF：

1. SSTF 是最短寻道时间优先算法。
2. 每次选择距离当前磁头位置最近的 I/O 请求作为下一次调度对象。
3. 优点是能减少平均寻道时间。
4. 缺点是可能使远离当前磁头的请求长期等待。

SCAN：

1. SCAN 是扫描算法，也称电梯算法。
2. 磁头沿一个方向移动，遇到 I/O 请求就处理。
3. 到达边缘或该方向无请求后，再反向扫描。
4. 它既考虑距离又考虑移动方向。
5. 它能克服 SSTF 可能产生饥饿的缺点。

计算方法：

1. 先写出各算法的访问顺序。
2. 再把相邻两次访问磁道号之差的绝对值相加。
3. SCAN 题目必须注意磁头初始方向。
4. 若题目未给方向，应先说明自己的方向假设。
